class Visitor(Generic[T]): (source)

View In Hierarchy

"Visitor" pattern abstract superclass implementation for tree traversals. Each class has corresponding methods, doing nothing by default; override individual methods for specific and useful behaviour. The `visit()` method is called by `walk()` upon entering a object. `walkabout()` also calls the `depart()` method before exiting a object. The generic methods call "``visit_`` + objet class name" or "``depart_`` + objet class name", resp. This is a base class for visitors whose ``visit_...`` & ``depart_...`` methods should be implemented for *all* concrete objets types encountered.
